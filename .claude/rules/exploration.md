# exploration avant modification

avant de modifier du code existant :

1. **comprendre le contexte**
   - grep/read pour voir comment c'est utilise
   - identifier les references et imports

2. **lister les impacts**
   - fichiers qui importent ce code
   - tests associes
   - documentation a mettre a jour

3. **planifier les changements**
   - modifier dans l'ordre des dependances
   - tester apres chaque etape
