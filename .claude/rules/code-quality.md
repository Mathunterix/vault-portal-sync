# qualité du code

## principes

- **DRY** : pas de duplication, extraire si répétition
- **complet** : pas de TODO, pas de placeholders, pas de "à implémenter plus tard"
- **lisibilité > performance** : code clair d'abord, optimiser si besoin prouvé
- **honnêteté** : si tu ne sais pas, dis-le au lieu d'inventer

## avant de coder

- vérifier si ça existe déjà (grep, composants existants)
- comprendre le contexte (lire les fichiers liés)

## le code doit être

- fonctionnel (pas de code mort)
- importé correctement (tous les imports nécessaires)
- typé (pas de `any`)
- testé si critique
